## How to Contribute

### Reporting Issues

Found a bug or have a suggestion?

1. **Check existing issues** - Search [Issues](../../issues) to avoid duplicates
2. **Create a new issue** - Click "New Issue" and describe:
   - What you found (bug, missing feature, unclear documentation)
   - Steps to reproduce (for bugs)
   - Expected vs. actual behavior
   - Your environment (OS, LaTeX distribution version, etc.)

### Improving Documentation

Documentation improvements are always welcome!

1. **Fork the repository** - Click "Fork" in the top-right corner
2. **Make your changes** - Edit `.md` files directly in your fork
3. **Test your changes** - Ensure formatting and links work correctly
4. **Submit a Pull Request** - Click "Pull Request" and describe your improvements

### Fixing Bugs or Adding Features

To contribute code changes:

1. **Fork the repository**
2. **Create a feature branch** - `git checkout -b fix/bug-name` or `git checkout -b feature/feature-name`
3. **Make your changes**
   - Keep changes focused and minimal
   - Test your changes locally
   - Follow the existing code style and formatting
4. **Commit with clear messages** - `git commit -m "Fix: description of what changed"`
5. **Push to your fork** - `git push origin fix/bug-name`
6. **Submit a Pull Request** - Describe what you fixed or added
